# **🧪 gitProject Prototypes and Experiments**

This directory is designated for all "Full Prototypes" and experimental code generated by volunteers using **AI-assisted tools** (Gemini, Co-pilot).

### **🎯 Folder Purpose**

The purpose of this folder is to store experimental code snippets, modules, and proofs-of-concept that are **not** yet ready for production integration. It acts as a mandatory holding area for all AI-generated code that is currently under review, verification, or study, ensuring the main codebase remains clean and audited.

### **⚠️ Governing Policy & Rules**

All content within this folder is governed by **VIBE CODE Policy (MOD\_SYS\_038), Section 3.5**.

1. **Isolation:** Prototypes must be fully contained within their own subdirectories (e.g., auth-flow-gemini-v1/).  
2. **Required Metadata:** Every subfolder **must** contain a README.md file using the **Prototype Metadata** structure to document the AI tool used, the objective, and known limitations.  
3. **Prompt Record:** Each prototype must link to one or more isolated prompt record files (as required by Policy Section 3.4).  
4. **No Direct Production Use:** Code in this directory must **never** be copied directly into the main application code without being thoroughly reviewed, verified, and manually tested.
